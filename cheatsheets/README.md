# cheatsheets/ — cikgu reference cards (grow on demand)

Small, single-topic cards cikgu points the student at during teaching. Two families, different
pedagogy (see `.claude/agents/cikgu.md` → "Troubleshooting vs Optimization"):

- `troubleshooting/` — diagnostic-search-under-uncertainty cards. Field shape:
  **Symptom / Trace / Root / Fix.** Impact tagged `[measured]` (real before→after cited) or
  `[projected]`. Fabricated metrics are the cardinal sin.
- `optimization/` — known-catalog pattern cards. Field shape: **What / Why / Applied / Effect.**
  Same honesty tags.

Cards are written on demand as real incidents/optimizations happen in the build — this starts
empty by design. cikgu MUST NOT invent a card from memory; no card yet → run the topic as a
hypothesis exercise instead. An interview drill sources its answer from a real card's fields,
never from recollection.
